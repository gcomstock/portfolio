<section id="pivot">

## Pivot

Early prototypes taught us what the data already suggested: the initial direction was too narrow. We needed to broaden scope, stagger the rollout, and fix the most painful UX gaps.

<FeatureBlock
  accent="blue"
  eyebrow="Design"
  title="Lo-fi mockups"
  media={<img src={imgPivLofi} alt="Lo-fi mockups" style={{width:'100%',height:'100%',objectFit:'cover',borderRadius:'4px'}} />}
>
  <p>Following the sprint, I partnered with engineering to make early single-page models to prove the troubleshooting journey end-to-end. It was a week of speed-sketching and iterating on core information architecture with the full team.</p>
</FeatureBlock>

<FeatureBlock
  accent="purple"
  reverse
  eyebrow="Design"
  title="Hi-fi prototypes"
  media={<img src={imgPivHifi} alt="Hi-fi prototypes" style={{width:'100%',height:'100%',objectFit:'cover',borderRadius:'4px'}} />}
>
  <p>Type prototyping got the team aligned on UI surfaces. Users could filter tag values using inline autocomplete, with suggestions limited to the previous 6 hours. Early feedback was strong — but validation revealed a gap.</p>
</FeatureBlock>

<FeatureBlock
  accent="pink"
  eyebrow="Reflection"
  title="Validation fell short"
  media={<img src={imgPivValid} alt="Validation fell short" style={{width:'100%',height:'100%',objectFit:'cover',borderRadius:'4px'}} />}
>
  <p>Building a front-end prototype against the existing API endpoints exposed the real problem: we could not design our way out of the underlying data model issues. External tools were already winning on this front. It became obvious we needed to pivot.</p>
</FeatureBlock>

<FeatureBlock
  accent="amber"
  reverse
  eyebrow="Strategy"
  title="Pivot points"
  media={<img src={imgPivPoints} alt="Pivot points" style={{width:'100%',height:'100%',objectFit:'cover',borderRadius:'4px'}} />}
>
  <p>Three pivots shaped the path forward: <strong>Broaden the appeal</strong> — open Explore to all data types so it looked like an established product from day one; <strong>Stagger the release</strong> — ship Metrics first, then Logs, then Traces, starting with Safety and Reliability by App; <strong>Fix the pain points</strong> — use a prioritized bug list from customer validation to resolve the highest-friction issues first.</p>
</FeatureBlock>

</section>

<section id="solution">

## Solution

We shipped in phases — Metrics GA, then Logs Beta, then Traces — validating the unified model and growing the user base before each new data type launched.

<FeatureBlock
  accent="blue"
  eyebrow="Phase 1 · Metrics"
  title="Metrics GA"
  media={<img src={imgSolMetA} alt="Metrics GA" style={{width:'100%',height:'100%',objectFit:'cover',borderRadius:'4px'}} />}
>
  <p>Once the core patterns of the new UX were established, I worked with engineering to make tight refinements almost daily. The first GA release unified metric querying — Builder or Manual mode, tag filtering, grouping, and multi-panel graph views all in one place.</p>
</FeatureBlock>

<FeatureBlock
  accent="purple"
  reverse
  eyebrow="Phase 1 · Metrics"
  title="Inline context and exploration"
  media={<img src={imgSolMetB} alt="Metrics inline exploration" style={{width:'100%',height:'100%',objectFit:'cover',borderRadius:'4px'}} />}
>
  <p>With data on screen, Explore offered inline jump paths to Logs or Traces without losing query context. Sidebar panels kept related metrics visible while a user drilled into a specific time series or anomaly.</p>
</FeatureBlock>

<FeatureBlock
  accent="green"
  eyebrow="Phase 1 · Metrics"
  title="Metrics in production"
  media={<img src={imgSolMetC} alt="Metrics in production" style={{width:'100%',height:'100%',objectFit:'cover',borderRadius:'4px'}} />}
>
  <p>The final Metrics GA interface shipped with a clean query builder, multi-series support, and deep linking. Engineers could land directly in an Explore view from a Slack alert or an Atlas dashboard without re-building their query.</p>
</FeatureBlock>

<FeatureBlock
  accent="amber"
  reverse
  eyebrow="Phase 2 · Logs"
  title="Logs Beta"

</FeatureBlock>

<FeatureBlock
  accent="blue"
  eyebrow="Phase 2 · Logs"

>
  <p>Error-level filtering, grouped log views, and Visualize mode let engineers move from "something is wrong" to a specific error message in seconds — all without leaving Explore or re-writing a Sumologic query.</p>
</FeatureBlock>

<FeatureBlock
  accent="purple"
  reverse
  eyebrow="Phase 3 · Traces"

</FeatureBlock>

</FeatureBlock>
